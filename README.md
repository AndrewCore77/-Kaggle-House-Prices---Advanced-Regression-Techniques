# House Prices Prediction

Проект по предсказанию стоимости недвижимости на датасете Ames Housing в формате соревнования Kaggle *House Prices: Advanced Regression Techniques*.

В репозитории собран полный базовый пайплайн для задачи регрессии: загрузка и анализ данных, обработка пропусков, кодирование признаков, масштабирование, обучение модели и оценка качества на отложенной выборке.

## О проекте

Цель проекта - предсказать `SalePrice`, итоговую цену продажи дома, на основе характеристик объекта: площади, качества отделки, района, года постройки, параметров гаража, подвала и других признаков.

В качестве базовой модели используется `LinearRegression`. Несмотря на простоту, такой подход уже даёт сильный и интерпретируемый baseline, на котором удобно показать весь процесс работы с табличными данными.

## Что сделано

- проведён первичный анализ целевой переменной `SalePrice`
- удалены экстремальные выбросы по цене
- выполнено разделение данных на train и test
- обработаны пропущенные значения для категориальных и числовых признаков
- категориальные признаки закодированы через `TargetEncoder`
- порядковые признаки переведены в числовой формат
- числовые признаки нормализованы с помощью `MinMaxScaler`
- обучена линейная регрессия
- рассчитаны метрики качества `RMSE` и `R2`
- дополнительно оценена стабильность модели через bootstrap-доверительные интервалы

## Результаты

Текущие результаты на отложенной выборке:

| Метрика | Значение |
| --- | ---: |
| RMSE train | 26,813 |
| RMSE test | 31,389 |
| R2 train | 0.8710 |
| R2 test | 0.8589 |
| RMSE 95% CI | 25,921 - 36,823 |
| R2 95% CI | 0.8248 - 0.8904 |

Краткая интерпретация:

- `RMSE` показывает среднюю величину ошибки прогноза в долларах
- `R2` показывает, какую долю вариации цен объясняет модель
- близкие значения на train и test говорят о том, что сильного переобучения в текущем baseline нет

## Как запустить

### 1. Клонировать репозиторий

```bash
git clone <your-repo-url>
cd <repo-folder>
```

### 2. Создать виртуальное окружение

```bash
python3 -m venv .venv
source .venv/bin/activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Запустить ноутбук

```bash
jupyter notebook notebooks/house-prices-linear-regression.ipynb
```

Ноутбук уже настроен на работу с данными из локальной папки `data/`, поэтому после клонирования проект можно открыть без правки путей.
